## Why

随着供应商数量增长，当前供应商管理页缺少搜索与汇总能力，管理员在查找站点、核对余额和识别异常站点时效率会快速下降。同时，系统还缺少供应商级可达性状态和按站点代理配置，导致无法在站点失联时及时摘除，也无法接入需要代理访问的上游站点。

## What Changes

- 为供应商管理页增加搜索能力，支持在大量站点中按名称、地址和相关字段快速定位目标供应商。
- 增加供应商余额汇总视图与 API，统计全部账户余额、余额更新时间状态及异常账户概况。
- 增加供应商级健康状态跟踪，展示最近可访问结果，并在站点不可访问时将其标记为不可用并停止参与站点使用流程。
- 增加按供应商单独开启的网络代理配置，使同步、签到和实际转发都可按站点走独立代理。
- 在管理端补充与上述能力匹配的可视化状态、筛选与配置入口，并保持敏感代理信息安全返回与导出行为。

## Capabilities

### New Capabilities
- `provider-management-discovery`: 支持供应商搜索、余额汇总和面向运营的管理概览能力。
- `provider-health-monitoring`: 支持供应商级站点可达性检测、状态展示和不可达时的自动摘除策略。
- `provider-proxy-routing`: 支持按供应商独立配置出站代理，并应用到同步、签到和实际上游转发链路。

### Modified Capabilities
- None.

## Impact

- Affected code:
  - 后端供应商查询、供应商模型、同步/签到/转发服务、Dashboard/Provider 管理 API
  - 前端供应商列表、供应商编辑表单、供应商概览卡片与状态展示
- Affected systems:
  - 供应商同步链路（pricing/tokens/balance）
  - 供应商签到链路
  - 真实请求转发链路
- Data/API impact:
  - 需要新增供应商健康与代理配置字段
  - 需要扩展供应商列表查询参数与新增供应商汇总/健康信息响应
- Security/operations:
  - 代理配置中的敏感信息需要避免在普通查询、导出和日志中泄漏
