# 添加供应商字段获取指南

本文说明“添加供应商”表单中几个关键字段如何获取或填写：`访问令牌`、`上游用户编号` 与 `站点代理`。

## Access Token

适用于“访问令牌”字段。

1. 登录你的上游 NewAPI 管理后台。
2. 进入个人设置或令牌页面，找到用于管理 API 的访问令牌。
3. 复制完整令牌字符串并填入网关“添加供应商”表单中的“访问令牌”。

注意事项：
- 不要在令牌前后添加空格或换行。
- 如果上游令牌轮换，请在网关中同步更新该字段。
- 建议仅授予必要权限，避免使用高危长期令牌。

## Upstream User ID

适用于“上游用户编号”字段（用于请求头 `New-Api-User`）。

1. 登录上游 NewAPI 后台。
2. 打开“个人信息 / 账号信息 / 用户信息”页面，找到用户 ID（通常是数字）。
3. 将该数字填入网关“上游用户编号”。

可选核验方法：
- 使用访问令牌请求上游 `GET /api/user/self`，确认返回中的用户标识与你填写的一致。

注意事项：
- 请填写数字 ID，不要填写用户名或邮箱。
- 若填错，可能导致同步、签到或创建上游令牌失败。

## Site Proxy

适用于“启用站点代理 / 代理地址”字段。

当上游站点无法从当前部署环境直连时，可以按供应商单独启用代理。

推荐填写格式：
- `http://proxy.example.com:7890`
- `http://user:password@proxy.example.com:7890`
- `socks5://127.0.0.1:1080`

注意事项：
- 代理地址必须包含协议头，例如 `http://` 或 `socks5://`。
- 代理配置按供应商单独生效，会同时影响同步、签到和实际上游转发。
- 编辑已有供应商时，如果代理地址输入框留空，系统会保持现有代理地址不变。
- 导出供应商配置时，系统会保留代理地址以便重新导入，请妥善保管导出的文件。

## 常见问题

### 看不到用户编号怎么办？
先检查当前账号是否有查看个人信息权限，或联系上游管理员确认账号信息页入口。

### Access Token 可否复用已有业务令牌？
建议使用专门用于网关对接的令牌，便于最小权限控制与后续轮换。
